# Zylo Instruments Site Guidelines

## Project
This is the official website for Zylo Instruments, an audio software company making VST instruments, effects, presets, and sound tools.

## Tech Stack
Use plain HTML, CSS, and JavaScript.
Do not convert the site to React, Next.js, Vue, or any framework unless explicitly requested.
Keep the code simple, readable, fast, and easy to edit.

## Brand Direction
Zylo Instruments should feel:
- premium
- minimal
- cinematic
- retro-modern
- experimental
- serious but not corporate
- designed for musicians, producers, sound designers, and ambient/experimental creators

## Visual Style
Use:
- black and warm dark gray backgrounds
- white, off-white, and soft gray typography
- subtle grain, dust, scratches, and faded analog texture
- spacious layouts
- Swedish/Japanese minimalism
- clean product sections
- strong contrast
- cinematic product imagery
- restrained animation

Avoid:
- generic SaaS gradients
- gamer RGB aesthetics
- cartoon graphics
- overly glossy 3D
- fake analog wood-panel plugin clichés
- cluttered layouts
- excessive neon

## Products
Main products may include:
- Medrone: flagship granular / ambient instrument
- Zylodelay: delay effect
- OrbitWeave: motion-modulation instrument/effect concept

## Homepage Goals
The homepage should:
- quickly explain what Zylo Instruments is
- make the brand feel premium and real
- feature Medrone as the flagship product
- include space for product demos
- include an email signup / early access CTA
- prepare for Lemon Squeezy or FastSpring checkout links later

## Copy Tone
Use concise, confident language.
Avoid hypey startup copy.
Avoid phrases like “revolutionary,” “game-changing,” or “next-gen.”
Prefer grounded phrases like:
- software instruments for motion, texture, and atmosphere
- sound tools for cinematic and experimental music
- designed for texture, movement, and mood

## Code Rules
- Keep file paths clean.
- Preserve existing assets unless asked to remove them.
- Make layouts responsive.
- Use semantic HTML where possible.
- Keep CSS organized by section.
- Do not add unnecessary dependencies.
- Do not break existing image paths.
- Do not remove working sections unless requested.

## Current Priority
Build a beautiful, simple, launch-ready landing page first.
Do not overbuild accounts, checkout, licensing, or backend systems yet.
